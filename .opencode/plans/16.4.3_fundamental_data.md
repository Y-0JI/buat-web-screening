# Plan 16.4.3 — Fundamental Data (Data Layer Murni)

## Konteks / Status Saat Ini

Fondasi data layer (16.4.1) dan Company Profile (16.4.2) sudah jadi. Untuk
16.4.3, sebagian besar infrastruktur **sudah ada** hasil pengerjaan sebelumnya:

- `app/providers/idx_provider.py::IdxProvider.fetch_fundamentals` → ambil dari
  IDX screener (market_cap, revenue, net_margin, pe, pbv, roa, roe, der,
  shares_outstanding, week52 change, dll)
- `app/providers/fundamentals_provider.py::FundamentalsProvider.fetch_fundamentals`
  → Yahoo (market_cap, pe, pb, dividend_yield, forward_pe, beta, 52w high/low,
  avg volume, shares_outstanding, dll)
- `app/repositories/fundamentals_repository.py::FundamentalsRepository` → merge
  IDX (primary) + Yahoo (gap-fill), cache 24 jam, normalisasi `pb`↔`pbv`
- `app/services/fundamentals_service.py::FundamentalsService` → validasi ticker +
  `get_fundamentals`
- Singleton wiring sudah ada di `app/providers/__init__.py`,
  `app/repositories/__init__.py`, `app/services/__init__.py`

**Yang BELUM ada untuk menuntaskan 16.4.3:**
1. Endpoint router `/api/stock/{ticker}/fundamentals` (mirip
   `stock_profile` di `app/routers/stock.py`).
2. Kanonikalisasi field ke daftar eksplisit 16.4.3 — saat ini nama field IDX &
   Yahoo masih acak (mis. `net_margin` vs `net_income`, `der` vs
   `debt_to_equity`) dan sebagian field 16.4.3 belum ter-cover.
3. Smoke test fundamental (arah 16.4.7) untuk memverifikasi field kanonik & fallback.

Scope disepakati: **data layer 16.4.3 murni — TIDAK menyentuh frontend
(Fundamental Card / Quick Stats Card tetap di 16.4.6).**

## Field Target (kanonik, dari dokumen 16.4.3)

Wajib ada (nama field kanonik): `market_cap`, `enterprise_value`, `pe`,
`forward_pe`, `pbv` (alias `pb`), `peg`, `eps`, `roe`, `roa`, `revenue`,
`revenue_growth`, `net_income`, `gross_margin`, `operating_margin`,
`debt_to_equity`, `current_ratio`, `free_cash_flow`, `dividend_yield`,
`dividend_payout_ratio`, `week_52_high`, `week_52_low`, `average_volume`,
`shares_outstanding`.

**Dropped (per dokumen):** `beta`, `book_value` → jangan dikanonikkan.
**Dipertahankan (konsumen 16.4.6):** `week_52_high`, `week_52_low`,
`average_volume`. `shares_outstanding` → HANYA di sini (bukan di profile).

## Langkah

### 1. Kanonikalisasi field di `FundamentalsRepository._merge_fundamentals`
- Bangun dict output dengan **key kanonik eksplisit** (whitelist) — bukan sekadar
  gabung mentah. Setiap key diisi dari sumber yang punya (IDX atau Yahoo),
  `None` bila tak ada.
- Pemetaan:
  - `market_cap` ← IDX `marketCapital` / Yahoo `marketCap` (sudah)
  - `pbv` (kanonik) ← IDX `pbv`; `pb` dipertahankan sebagai alias (sudah)
  - `debt_to_equity` ← IDX `der` / Yahoo `debtToEquity`
  - `net_income` ← Yahoo `netIncomeToCommon` (IDX hanya `net_margin` → boleh
    dihitung `net_income ≈ revenue * net_margin` bila ada keduanya)
  - `revenue` ← IDX `tRevenue` / Yahoo `totalRevenue`
  - `revenue_growth` ← Yahoo `revenueGrowth`
  - `eps` ← Yahoo `trailingEps` / `forwardEps`
  - `peg` ← Yahoo `pegRatio`
  - `enterprise_value` ← Yahoo `enterpriseValue`
  - `gross_margin` ← Yahoo `grossMargins`
  - `operating_margin` ← Yahoo `operatingMargins`
  - `current_ratio` ← Yahoo `currentRatio`
  - `free_cash_flow` ← Yahoo `freeCashflow`
  - `dividend_payout_ratio` ← Yahoo `payoutRatio`
  - `week_52_high` ← Yahoo `fifty_two_week_high`
  - `week_52_low` ← Yahoo `fifty_two_week_low`
  - `average_volume` ← Yahoo `avg_volume_3m`
  - `shares_outstanding` ← IDX securities / Yahoo `sharesOutstanding` (sudah)
  - `roe`, `roa`, `pe`, `dividend_yield`, `forward_pe` ← sudah ada di IDX/Yahoo
- Beta & book_value **tidak** dimasukkan ke whitelist kanonik.
- `source` = `"idx+yahoo"` / `"idx"` / `"yahoo"` sesuai yang berhasil (sudah).
- `fetched_at` dipertahankan.

### 2. Tambah endpoint di `app/routers/stock.py`
- Tambah `FundamentalsResponse(BaseModel)` (success, ticker, data, error) — tiru
  `CompanyProfileResponse`.
- Tambah `@router.get("/{ticker}/fundamentals", response_model=FundamentalsResponse)`
  yang memanggil `fundamentals_service.get_fundamentals(clean)` dengan handler
  error persis seperti `stock_profile`.

### 3. Smoke test fundamental (`test_providers_smoke.py`)
- Tambah `test_fundamentals_endpoint_wiring` → assert path
  `/api/stock/{ticker}/fundamentals` ada di `stock_router.routes`.
- Tambah `test_fundamentals_fields` (async) → panggil
  `FundamentalsRepository.get_fundamentals("BBCA")`; bila tidak error, assert
  key kanonik wajib hadir (setidaknya yang punya consumer nyata: `market_cap`,
  `pe`, `pbv`/`pb`, `roe`, `roa`, `week_52_high`, `week_52_low`,
  `average_volume`, `shares_outstanding`). Jika offline/error, cukup assert
  kembalian dict aman dengan key `error` (tidak raise) — konsisten pola
  `_test_idx_fallback`.
- Jalankan `python3 test_providers_smoke.py` untuk pastikan lolos.

## Catatan / Keputusan
- **Tidak tambah dependensi baru.** Semua field diambil dari IDX screener &
  Yahoo `stock.info` yang sudah dipakai — hindari pemanggilan endpoint/proses
  ekstra.
- `net_income` dihitung dari `revenue * net_margin` sebagai fallback ringan bila
  Yahoo tidak memberi `netIncomeToCommon` (ponytail: estimasi, bukan field
  presisi GAAP; cukup untuk konsumsi AI awal).
- Field yang IDX/Yahoo sama-sama tidak punyai (mis. `enterprise_value`,
  `peg`, `free_cash_flow`, `current_ratio`, `dividend_payout_ratio`) akan
  `None` — downstream (16.4.6/16.8) yang memutuskan fallback, bukan di sini.
- Frontend (Fundamental Card, Quick Stats Card) **tidak** dikerjakan di sini.

## Acceptance (arah 16.4.7)
- Endpoint `/api/stock/{ticker}/fundamentals` responsif & konsisten error
  handling dengan `/profile`.
- Output repo adalah dict dengan key kanonik 16.4.3 (tanpa `beta`/`book_value`).
- Smoke test lolos; tidak ada error pada import/wiring.
