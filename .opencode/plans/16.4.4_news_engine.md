# Plan 16.4.4 — News Engine (Raw Aggregator)

> Revisi per review notes: gunakan respons nyata provider (#1), sertakan
> `source` tiap item (#2), dedupe lintas-sumber lebih kuat (#3), `related_ticker`
> selalu konsisten (#4), validasi diperluas (#5). Tetap scope 16.4.4 — frontend,
> sentiment, AI processing ditunda (16.4.6 / 16.8).

## Konteks / Status Saat Ini

Dari eksplorasi kode (`backend/app/...`):

- `providers/news_provider.py::NewsProvider.fetch_news` → Yahoo Finance
  (`stock.news`), field: `title`, `publisher`, `link`, `published`, `summary`.
- `providers/rss_provider.py::RssProvider.fetch_rss` → Google News RSS, field:
  `title`, `publisher`, `link`, `published`, `summary`, `related_ticker`.
- `repositories/news_repository.py::NewsRepository` → **hanya pakai Yahoo**,
  TTL 15 menit. `RssProvider` ada tapi **belum di-wire** ke repository.
- `services/news_service.py` + `routers/news.py` → endpoint
  `GET /api/stock/{ticker}/news`, schema `NewsItem(title, publisher, link,
  published, summary)`.

**Gap vs dokumen 16.4.4:** (1) IDX News (primary) belum ada; (2) tidak ada
agregasi multi-source; (3) field belum kanonik (`headline`/`published_date`/
`url`/`related_ticker`); (4) TTL 15 mnt ≠ arah 16.4.5 (News = 1 jam).

## Langkah

### 0. Verifikasi respons nyata SEBELUM mapping (Review #1)
- Sebelum tulis parser, ambil respons mentah sekali lewat `curl`/Python REPL
  (bukan di-test):
  - IDX: `GET .../GetProfileAnnouncement?KodeEmiten=BBCA&...` → catat struktur
    aktual (nama array, field `JudulPengumuman`/`TglPengumuman`/lampiran).
  - Yahoo: `yf.Ticker("BBCA.JK").news[0]` → catat struktur `content` aktual.
  - RSS: ambil 1 item `<item>` → catat elemen aktual.
- Parser di tiap provider **mengikuti** struktur tervalidasi ini. Bila IDX
  ganti struktur, cukup ubah parser di provider tsb (satu tempat). Tidak ada
  asumsi field hardcode tanpa cross-check ke respons nyata.

### 1. `IdxProvider.fetch_company_announcements` (primary)
- Method di `idx_provider.py`, pakai `_fetch_json` + `_BROWSER_HEADERS` (sama
  pola `fetch_company_profile`). Return `{"items": [...], "fetched_at": ...}`
  atau `{"error": ...}`.
- Mapping mengikuti hasil Langkah 0; field kanonik: `headline`, `publisher`
  ("IDX"), `published_date`, `summary`, `url` (lampiran pertama), `related_ticker`
  (diisi dari `code`), `source` = `"IDX"`.
- Fallback aman: return `{"error": ...}` bila gagal (konsisten provider lain).

### 2. Normalisasi field + `source` (Review #2)
- Helper `_normalize_item(raw, source, ticker)` di `news_repository.py`:
  - IDX/Yahoo/RSS → ke-7 field kanonik. `title→headline`, `link→url`,
    `published→published_date`.
  - `related_ticker` SELALU = `ticker` (Review #4), bukan dari provider.
  - `source` diisi eksplisit per pemanggil ("IDX"/"Yahoo"/"Google RSS").
  - Field hilang → string kosong (tidak None) agar schema seragam.

### 3. Agregasi + dedupe kuat di `NewsRepository` (Review #3)
- Terima 3 provider: `IdxProvider` (primary), `NewsProvider` (Yahoo),
  `RssProvider` (RSS). Default instance seperti repository lain.
- `get_news(symbol, limit)`:
  1. Cek cache (`ticker:limit`).
  2. Fetch ke-3 sumber (tiap sumber di-try sendiri; satu gagal tidak
     menggagalkan lain → kumpulkan `items` + catat error per sumber).
  3. **Dedupe lintas-sumber yang kuat**: key dedupe = hash normalisasi
     `headline` (lowercase, buang tanda baca & whitespace) sebagai primer;
     bila headline sama persis → satu saja. Sekunder: cocokkan `url` (normalize
     domain/path) bila headline berbeda tapi URL merujuk berita sama. Ini tetap
     bersih walau URL tiap sumber beda (Review #3).
  4. Sort by `published_date` desc (parse tanggal; item tanpa tanggal di akhir).
     Potong `limit`.
  5. Cache hasil (TTL 1 jam).
- Semua sumber error → return `{"error": "...", "fetched_at": null}`.

### 4. Schema & Router
- `schemas/news.py::NewsItem`: `headline`, `publisher`, `published_date`,
  `summary`, `url`, `related_ticker`, `source`. (`title/link/published` →
  diganti namanya.)
- `routers/news.py`: sudah pakai `NewsItem` → ikut berubah; signature endpoint
  tetap.

### 5. TTL cache → 1 jam
- `_NEWS_TTL = 3600` (arah 16.4.5).

### 6. Smoke test diperluas (Review #5)
- `test_news_repo_wiring` → assert `news_repository` punya `_idx_provider`
  (IdxProvider), `_provider` (NewsProvider/Yahoo), `_rss_provider` (RssProvider).
- `test_news_normalize_fields` → `_normalize_item` untuk contoh Yahoo & RSS:
  assert ke-7 field kanonik ada, `related_ticker` == ticker, `source` benar.
- `test_news_dedupe_and_sort` (sync) → beri items duplikat lintas sumber
  (headline sama, url beda) → hasil dedupe jadi 1; urut desc by date.
- `test_news_fallback` (async, mock `_fetch_json` jadi boom + Yahoo/RSS error)
  → repository **tidak raise**; kembalikan dict aman (`error` atau items kosong)
  dengan `source` tercatat.
- Jalankan `python3 test_providers_smoke.py` → semua lolos.
