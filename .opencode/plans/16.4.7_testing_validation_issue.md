# 16.4.7 — Testing & Validation

Validasi lapisan Data Intelligence 16.4.x sebelum dianggap selesai. Ikuti 6
sub-bagian roadmap:

1. **API Test** — endpoint `GET /api/research` (ada `fundamentals`, `news`,
   `ai_available`), `/api/stock/{ticker}/fundamentals`, `/api/stock/{ticker}/news`
   respons & bentuk data benar. Pakai FastAPI TestClient bila httpx ada, atau
   smoke panggil handler langsung / curl.
2. **Cache Test** — `app/cache/*` (ttl, memory_cache, service, technical_cache):
   TTL expire, `clear(prefix)`, `cache_service` singleton, `POST /api/cache/refresh`
   (global & `?type=`) tidak error. Smoke panggil fungsi langsung.
3. **Error Handling Test** — provider (Idx/Rss/Yahoo) tidak raise saat jaringan
   gagal (kembalikan dict `error` / `(None, False)`); endpoint tetap 200 &
   `ai_available=false` saat gagal, tidak ada exception bocor.
4. **Frontend Rendering Test** — `npx tsc --noEmit` + `npm run build` lolos
   (kartu Fundamental/Quick Stats/Profile/News 16.4.6). Opsional manual dev server.
5. **Performance Test** — ukur latency research (cache warm vs cold); teknikal
   scoring 60s tidak dihitung ulang tiap request. Smoke timing print durasi.
6. **Await Coverage Test** — audit semua `await` provider/repository/DB: pastikan
   tidak ada call blocking sync & tidak ada coroutine tidak di-await. Trigger jalur
   async & pastikan selesai tanpa warning "coroutine never awaited".

**Konvensi:** ikuti gaya `test_providers_smoke.py` (file `test_*.py`, jalan via
`python3`, print + exit non-zero bila gagal). Jangan tambah pytest kecuali
diminta. Frontend tidak tambah dependency. **JANGAN ubah logika produksi** — hanya
tambah file test; bila nemukan bug, catat di issue (PR terpisah).

**Verifikasi:** `python3 backend/test_16.4.7_*.py` lolos; `npx tsc --noEmit &&
npm run build` lolos.

**Deliverable:** Branch `feat/16.4.7-testing-validation` dari `main`; PR ke
`main` (jangan di-merge); issue GitHub isi plan ini.
