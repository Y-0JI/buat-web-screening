# 16.4.8 — Dashboard & Screening Validation

Pastikan screening & dashboard mencerminkan data lengkap, konsisten, mudah
dipahami. Screening harus pakai universe saham nyata (bukan sample).

**Temuan audit:**
- `GET /api/screen` (`routers/screening.py`) pakai `list(MOCK_DATA)` → hanya ~15
  ticker sample, bukan universe IDX. (langgar scope #1)
- `run_batch_scan` (scheduler) sudah pakai `get_listed_tickers()` (universe nyata
  ~955). Endpoint on-demand jadi TIDAK konsisten dengan scheduler. (scope #2)

**Langkah high-level:**
1. Universe screening — `GET /api/screen` ambil ticker dari `get_listed_tickers()`
   (sama dengan scheduler), bukan `MOCK_DATA`.
2. Konsistensi scoring — endpoint & scheduler pakai jalur scoring sama (mode, cache).
3. Tampilan dashboard — tinjau komponen (AI picks, market overview, watchlist,
   riwayat) agar relevan, tidak membingungkan, label jelas.
4. Riwayat aktivitas — tampil konsisten (ticker, skor, waktu).
5. Navigasi & aksi — fungsi tiap tombol jelas, tidak ke halaman kosong.

**Batasan:** perubahan minimal & fokus (sumber universe + konsistensi); review UI
tanpa redesign besar; tidak ubah logika scoring/funnel.

**Verifikasi:** `/api/screen` kembalikan saham jauh >15 & konsisten dengan
scheduler; `npx tsc --noEmit && npm run build` lolos; review dashboard.

**Deliverable:** Branch `feat/16.4.8-dashboard-screening-validation` dari `main`;
PR ke `main` (jangan di-merge); issue GitHub isi plan ini.
