# 16.4.7 — Testing & Validation

## Tujuan
Validasi seluruh lapisan Data Intelligence 16.4.x (fundamental, news, smart
cache, research, error handling, frontend) sudah bekerja & aman sebelum
dianggap selesai. Ikuti sub-bagian roadmap: API Test, Cache Test, Error
Handling Test, Frontend Rendering Test, Performance Test, Await Coverage Test.

## Konteks Repo
- Pola test saat ini: **smoke test tanpa framework** (jalankan via
  `python3 backend/test_xxx.py`, lihat `test_providers_smoke.py`). Tanpa pytest
  terkonfigurasi & tanpa httpx di requirements.
- Frontend: tidak ada framework test (hanya `npx tsc --noEmit` + build).
- Target sudah ada: `app/routers/{research,news,cache,stock}.py`,
  `app/cache/*`, `app/repositories/*`, `app/providers/*`, komponen
  `frontend/src/components/renderers/*`.

## Rencana High-Level (6 sub-bagian)

### 1. API Test
- Verifikasi endpoint utama 16.4.x merespons & bentuk datanya benar:
  - `GET /api/research?ticker=BBBCA` → ada `fundamentals` (field kanonik),
    `news` (headline/url/published_date/source), `ai_available`.
  - `GET /api/stock/{ticker}/fundamentals` → field kanonik 16.4.3 tidak None
    bila sumber ada.
  - `GET /api/stock/{ticker}/news` → list NewsItem kanonik.
- Bila httpx/TestClient tersedia: pakai FastAPI TestClient. Bila tidak: smoke
  via import router & panggil fungsi handler secara langsung (atau jalankan
  server lokal & curl). Putuskan agent saat eksekusi — yang penting terverifikasi.

### 2. Cache Test
- `app/cache/*` (ttl, memory_cache, service, technical_cache):
  - Set/get dengan TTL → kadaluarsa sesuai.
  - `clear(prefix)` menghapus hanya prefix terkait.
  - `cache_service` singleton terinisialisasi; `POST /api/cache/refresh`
    (global & `?type=`) tidak error.
- Jalankan via smoke script panggil fungsi cache langsung.

### 3. Error Handling Test
- Provider (IdxProvider, RssProvider, Yahoo via repository) tetap tidak raise
  saat jaringan gagal → kembalikan dict `error` / `(None, False)`.
- Endpoint research/news tetap 200 & `ai_available=false` saat AI/provider
  gagal (sudah di 16.4.x). Verifikasi tidak ada exception bocor ke response.

### 4. Frontend Rendering Test
- `cd frontend && npx tsc --noEmit` lolos (kartu 16.4.6: Fundamental, Quick
  Stats, Profile, News).
- Build sukses (`npm run build`) tanpa error tipe.
- (Manual, opsional) jalankan dev server, buka riset BBCA → kartu tampil dengan
  data; field None tidak tampil aneh.

### 5. Performance Test
- Ukur latency endpoint research (cache warm vs cold) — pastikan cache 16.4.5
  menurunkan waktu respons berarti.
- Technical scoring (technical_cache 60s) tidak dihitung ulang tiap request.
- Cukup smoke timing (print durasi), bukan benchmark berat.

### 6. Await Coverage Test (async DB & provider call)
- Audit semua `await` pada call provider/repository/DB: pastikan tidak ada
  call blocking (sync) yang seharusnya async, dan tidak ada coroutine
  tidak-di-await (typo `foo()` vs `await foo()`).
- Jalankan smoke yang men-trigger jalur async (research → repository →
  provider) dan pastikan selesai tanpa warning "coroutine never awaited".
